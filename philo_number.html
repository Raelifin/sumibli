The Standard Framework

Let's consider the standard Sumibli numeric framework.

Consider â€œsipaâ€. This does not mean â€œperson numberâ€. We forget that â€œpaâ€ means â€œpersonâ€, and see it as the letters â€œpâ€ and â€œaâ€. Here's where things get really different: the standard Sumibli numerical framework doesn't use base-ten, it uses base-six. The digits â€œ22â€ thus mean two-ones and two-sixes, ie fourteen.

Even more divergent from English, the standard numerical framework is little-endian. This mean the least significant digits in the number come first, followed by more significant digits. This means â€œsibataâ€ corresponding to 32 means three-ones and two-sixes, ie fifteen.

Little-endian is more correct because we don't know how many digits will occur, so we can immediately start hearing the number instead of having to remember early digits and be surprised when we hear/read a new digit. For example, let's say I read â€œsiga...â€. If this is little-endian, then I can be confident that I'm reading a number that has a 5 in the one's-place, whereas if I'm big endian than I know that there's a five in the most significant digit slot, but I need to see the whole number before I can figure out what place that actually is.

(One hint that little-endian is the correct way of writing numbers is that the â€œ0,1,2...â€ digits are Arabic in origin, and Arabic is read right-to-left.)

But the fun doesn't stop here. Remember that we can annotate our digits with vowels. Thus â€œsitaâ€ is not the same as â€œsitiâ€. What does â€œsitiâ€ mean? Remember that we're still in base-six, however, so .2 corresponds to two-sixths or one-third.

Unlike when annotating with â€œaâ€, standard fractional digits are big-endian. This is because of the exact same logic as before. Seeing a prefix of a number should not significantly change the number. If we know what number place we're reading, big endian is superior because we usually care most about the most significant digit. In a non-fractional number we don't have that luxury, but in a fractional number we do!
