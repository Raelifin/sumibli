Prerequisites

This document will include definitions written in a simple formal language. Each definition consists of a name for the thing being defined, then a â€œâ‰”â€ symbol, then the structure of the thing being defined.

The â€œ|â€ symbol indicates that the structure can either be the thing on the left of the pipe or the thing on the right. Multiple â€œ|â€ symbols indicates multiple options. Each option is called â€œa constructorâ€, because it describes a way of building an instance of the named thing. Constructors are distinct even though they're not named (in this document).

The â€œ&â€ symbol indicates the structure consists of both the thing on the left of the ampersand and the thing on the right. Multiple â€œ&â€ symbols indicate a structure with many parts. Ampersands have a higher precedence than pipes, meaning expressions like a | b & c should be read as a | (b & c).

Variables that define families of definitions are upper-case single-letters. The phrase â€œoptional Xâ€ refers to a structure that can either be empty or can have a single X in it. We'll also use â€œX?â€ as a synonym for â€œ(optional X)â€. The term â€œbitâ€ (aka â€œboolâ€) refers to a structure that can either be â€œloâ€ or â€œhiâ€. The phrase â€œenforced X Bâ€ refers to a structure that can either have a single X, or if B = lo, can also be empty. The phrase â€œlist Xâ€ refers to a structure that consists of zero or more X's in some order. The phrase â€œlist1 Xâ€ is a non-empty list of X's.

A â€œfancy-list Xâ€ takes a family of structures (X) that's parameterized on two bits. The fancy-list contains zero or more items, just like a normal list. Each item in the fancy-list is an X, where the first bit is lo iff the X is the first item and the second bit is lo iff it's the last item. Thus, a fancy list with four items would be [X lo hi, X hi hi, X hi hi, X hi lo].

The syntax â€œA : Bâ€ indicates that we're giving the name â€œAâ€ to the B structure for the sake of discussing it. If an equality (or inequality) shows up in a structure, it is an unspoken/unwritten constraint on that constructor. For instance, â€œ(A : bit) & (B : bit) & (A = B)â€ means we have two bits that must be equal.

Words

A word (â€œimaiâ€) is either a heavy-defined-word (â€œimaizladuâ€), an emotion-word (â€œimaipauâ€), or a strange-word (â€œimaivlaâ€).

imai â‰” imaizladu | imaipau | imaivla

A heavy-defined-word is a defined-word and a thur*. Defined-words (â€œimaizlaâ€) are parameterized on a bit, and in our heavy-defined-word that bit is lo.

Imaizla are built up in sections from left-to-right. The leftmost section consists of a standard consonant and a middle-sound. We can bolt on an additional section using a bridge-consonant, a standard consonant, and a middle-sound. Finally, we can bolt a section that consists of an entire imaizla by using a thur* and a â€œthijâ€. In this last case we can explicitly end the section by â€œthijâ€ & â€œthuiâ€, and this explicit end is required when the bit-parameter is hi.

  â‰” ufamglanaiva & ufamglabai

  | imaizla hi & thur* & thij
      & imaizla lo & enforced (thij & thui) B

Foreign Text

Foreign concepts can be imported into Sumibli by using a â€œstrangerâ€ (â€œivlaâ€) structure. This consists of a â€œ-thâ€ suffix, that is (unlike all other instances of â€œthuxâ€) pronounced unvoiced, followed by some arbitrary sounds. Alas, because foreign text can have arbitrary sounds, there is no explicit marker for the end of foreign speech; it's up to the speaker to make it clear that they're speaking Sumibli again. When writing, it's customary to include spaces around foreign words to distinguish them.

An ivla can also be used to quote a Sumibli speaker. Like with foreign text, this is marked using an unvoiced â€œthuxâ€. It is strongly recommended (but not required) for quoted expressions to explicitly end using â€œtshâ€. When writing quotes, quote marks are often used (sometimes in combination with whitespace).

Concepts

A concept (â€œumaiâ€) is built out of either a heavy-defined-word (â€œimaizladuâ€), a stranger (â€œivlaâ€), or a name (â€œimaivaâ€).

Names are either proper nouns built from â€œthaâ€ and some imaizla, or are pronouns formed from the other cases of the last imaivla constructor.

Sentences and Expressions

A sentence (â€œidamiâ€) is a fancy-list of expression-peices (â€œublirmiâ€) preceded by a semi-optional sentence-start (â€œidamituâ€). Like ublirmi, idami are also parameterized on two bits.

Sentence-starts (â€œidamituâ€) are the letters thit and thij followed by any diphthong.

An expression (â€œimiâ€) is an orientation followed by a fancy-list of sentences (â€œidamiâ€) and terminated by an optional â€œtshâ€. We are allowed to drop the orientation if there is at least one thur with an explicit orientation before the first imaizladu. Unlike everywhere else, the formal definition doesn't include this rule (because it's annoying to formally specify).
